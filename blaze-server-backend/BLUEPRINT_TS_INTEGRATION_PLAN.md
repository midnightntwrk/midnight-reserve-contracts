# Blueprint TypeScript Integration Plan

## Overview

This plan outlines the integration of TypeScript classes generated by the `@blaze-cardano/blueprint` compiler into the demo system to provide type safety, better developer experience, and reduced errors in contract interactions.

## Goals

1. **Type Safety**: Compile-time validation of datum/redeemer structures
2. **Developer Experience**: IntelliSense, autocomplete, and contract-specific methods
3. **Error Reduction**: Automatic serialization and validation
4. **Maintainability**: Contract evolution with type safety
5. **Backward Compatibility**: Maintain existing generic monadic functions

## Current State Analysis

### What We Have
- Generic monadic functions (`lockToContract`, `unlockFromContract`)
- Manual datum/redeemer construction
- Raw blueprint JSON parsing
- Basic error handling

### What We're Missing
- Type-safe datum/redeemer construction
- Contract-specific methods
- Automatic CBOR serialization
- IntelliSense and autocomplete
- Compile-time validation

## Proposed Architecture

### 1. TypeScript Class Generation
```
blueprint.json → @blaze-cardano/blueprint → TypeScript classes
```

### 2. Contract-Specific Monadic Functions
```typescript
// Instead of generic functions
await lockToContract(scriptHash, { datum: 42, ... });

// Contract-specific with type safety
await HelloWorldContract.lock({ datum: HelloWorldDatum.fromData(42), ... });
```

### 3. Hybrid Approach
- Keep existing generic functions for backward compatibility
- Add type-safe contract-specific functions
- Allow gradual migration

## Detailed Implementation Plan

### Phase 1: Isolated Spikes (Showstopper Investigation)

#### Spike 1: TypeScript Class Generation ✅ COMPLETED
**Goal**: Verify we can generate and use TypeScript classes from blueprint JSON

**Tasks**:
1. ✅ Install `@blaze-cardano/blueprint` package
2. ✅ Generate TypeScript classes from `./plutus.json`
3. ✅ Create simple test script to verify class functionality
4. ✅ Test datum/redeemer construction and serialization

**Results**:
- ✅ **TypeScript Generation**: Successfully generates TypeScript files from `plutus.json`
- ✅ **TypeScript Compilation**: Can compile (with proper environment setup)
- ✅ **Script Classes**: Provides CBOR-encoded scripts for transactions
- ❌ **Easy Construction**: No `MyDatum.fromData(42)` or `MyDatum({ thing: 42n })` helpers
- ❌ **CBOR Serialization**: No `datum.toCbor()` methods on constructed objects
- ❌ **Validation Helpers**: No easy validation or error handling for datum construction

**Key Findings**:
- Generated code focuses on **script construction**, not **data construction**
- `@blaze-cardano/data` Type system creates **type definitions**, not **constructors**
- Manual construction still required: `{ thing: 42n, ctor: 0n }`
- No easy construction helpers like `MyDatum.fromData(42)`
- No CBOR serialization methods on datum objects

**Conclusion**: **DO NOT PROCEED** - The blueprint TypeScript integration does NOT provide the easy construction helpers we want.
